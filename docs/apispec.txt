Artika API Specification

Users - There are two types of users, a 'designer' and a normal 'user'. I will refer to normal users as 'user' and designers
as 'designer'. These accout types come with seperate features because they are so distinct, but the designer account is a super
set of a user account.

API:

User management

Create a user

POST /user
	fullName - string
	address - string
	isDesigner - boolean
	oAuth2Id - string
returns
	message|error - string

Delete a user 

DELETE /user
	session - string
returns
	message|error - string	

Edit fields of a users profile. They all have to be valid changes for
this to actually apply the update.

PATCH /user
	session - string
	fields - string[]
	values - string[]
returns
	message|error - string

Create a user session from an oauth2id. 

GET /session
	oAuth2Id - string
returns
	session - string	
	message|error - string

Delete a session.

DELETE /session
	session - string
returns
	message|error - string
